Quel messaggio “Failed to start embedded python interpreter!” significa una cosa precisa:
➡️ l’EXE non riesce ad avviare il Python “interno” che PyInstaller ha impacchettato (quello che viene estratto in Temp/_MEI...).
Quindi non è un errore del tuo codice, è un problema di build / librerie mancanti / antivirus / file corrotti.
✅ Cause più comuni (e come risolvere)
1) Antivirus / Windows Defender blocca l’estrazione in TEMP (molto frequente)
PyInstaller estrae tutto in: C:\Users\...\AppData\Local\Temp\_MEIxxxxx
Se Defender blocca anche solo 1 file (tipo python3.dll), l’app non parte.
✅ Soluzione:
Disattiva temporaneamente Defender (solo per test)
Oppure aggiungi esclusione cartella:
la cartella dove tieni l’exe
e anche C:\Users\chiara\AppData\Local\Temp
📌 Prova anche: tasto destro → Proprietà → “Sblocca” (se c’è).
2) Hai compilato con Python 3.14 (troppo nuovo / instabile per PyInstaller)
Dai tuoi log prima si vedeva Python 3.14.2. PyInstaller spesso dà problemi con versioni troppo nuove.
✅ Soluzione consigliata (la più efficace): 👉 compila con Python 3.11 o 3.12
Questa è la combo più stabile:
Python 3.11.x
PyInstaller ultimo
Windows 10/11
3) Mancano runtime Microsoft Visual C++ (VC Redist)
Se sul PC dove lo apri manca il runtime, Python embedded non parte.
✅ Soluzione: Installa:
Microsoft Visual C++ Redistributable 2015–2022 (x64)
(è una delle cause più comuni su PC “puliti”)
4) EXE “onefile” corrotto o bloccato
Se l’exe è stato scaricato male o Windows lo blocca, succede.
✅ Soluzione:
riscarica
rinomina semplice tipo Pickfair.exe
mettilo in una cartella tipo C:\Pickfair\ (non Desktop, non cartelle protette)
5) Build sbagliata (file/dll non inclusi)
Se hai DLL esterne (tipo OpenSSL, Telethon, cryptg, ecc.) e PyInstaller non le prende, può crashare prima di partire.
✅ Soluzione: Per test fai build in modalità “cartella”:
Copia codice
Bash
pyinstaller main.py --onedir --noconsole
Se in onedir parte ma in onefile no → è colpa dell’estrazione TEMP / antivirus.
🔥 Test rapido per capire subito la causa
Compila così:
Copia codice
Bash
pyinstaller main.py --onedir --console
Poi avvia l’exe da terminale. Se crasha, vedrai l’errore vero (DLL mancante ecc).
