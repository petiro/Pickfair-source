🎯 CAUSA PIÙ PROBABILE (90%)

Stai chiamando send_message() da un thread diverso da quello dove gira l’event loop di Telethon, oppure l’evento che “rileva la scommessa” viene triggerato più volte o zero volte.

Quindi:

o non viene mai inviato

o viene “tentato” ma fallisce in silenzio

✅ SOLUZIONE CORRETTA (TESTATA)
1️⃣ Devi avere UN SOLO event loop Telethon

E devi salvarlo quando lo avvii.

telegram_loop = asyncio.new_event_loop()

def start_telegram():
    asyncio.set_event_loop(telegram_loop)
    telegram_loop.run_until_complete(client.start())
    telegram_loop.run_forever()

threading.Thread(target=start_telegram, daemon=True).start()


⚠️ Non creare loop diversi

2️⃣ send_message DEVE essere così (OBBLIGATORIO)
def send_telegram_message(chat_id, text):
    future = asyncio.run_coroutine_threadsafe(
        _send(chat_id, text),
        telegram_loop
    )

    try:
        future.result(timeout=15)
    except Exception as e:
        logger.exception("Errore invio Telegram")

async def _send(chat_id, text):
    if not client.is_connected():
        await client.connect()

    entity = await client.get_entity(chat_id)
    await client.send_message(entity, text)


📌 Senza get_entity() → spesso NON invia
📌 Senza future.result() → errori silenziosi
